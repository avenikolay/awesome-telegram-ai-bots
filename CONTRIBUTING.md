# Contributing

Thanks for suggesting an addition.

## Before you open a PR

- **Category fit** — place the entry in the most specific category. Don't duplicate across categories.
- **Production-grade** — the project should have real users or a meaningful community. No single-commit demos or abandoned repos.
- **One-line format** — `**[Name](url)** — what it does and one concrete technical detail (stack, pricing, scale).`
- **Canonical URL** — link to the official site or main repo, not a mirror.
- **Stable ordering** — new entries go at the end of their category unless they clearly outrank an existing one.

## Good example

```markdown
- **[aiogram](https://github.com/aiogram/aiogram)** — modern, fully asynchronous Python framework for Telegram Bot API. First-class support for FSM, middleware chains, webhook + polling.
```

## Bad examples

- `- [Cool bot](url) — it's cool` (no technical detail)
- `- [My project](url)` (no description at all)
- Adding the same entry to 3 categories
- Adding your own unreleased project

## Commit style

One entry per commit is ideal. Keep commit messages descriptive:

```
Add HoneyChat to Production AI Bots
```
